# n8n Workflow Implementation for AI Predictive Maintenance

## Overview
This document provides complete n8n workflow configurations for automating predictive maintenance processes based on AI health state predictions.

## API Integration
**FastAPI Service:** `http://localhost:8000`

### Available Endpoints:
1. **Automation Trigger**: `/automation/trigger` - Main decision point
2. **Health Monitoring**: `/equipment/health` - Equipment status
3. **Alert Management**: `/automation/alerts` - Detailed alert information
4. **Webhook Health Check**: `/n8n/webhook/health-check` - Simplified status

## Process Automation Logic

### Decision Tree:
1. **Every 15 minutes**: Check `/automation/trigger`
2. **If trigger_automation = true**:
   - Check alert_level
   - Execute appropriate workflow
3. **Alert Level Actions**:
   - **CRITICAL**: Emergency protocol (Slack + Email + ServiceNow)
   - **HIGH**: Urgent maintenance (Email + Work order)
   - **MEDIUM**: Preventive scheduling (Email notification)
   - **LOW**: Continue monitoring

### Environment Variables for n8n:
```bash
# API Configuration
PREDICTIVE_MAINTENANCE_API_URL=http://localhost:8000
API_CHECK_INTERVAL=15  # minutes

# Notification Configuration  
SLACK_WEBHOOK_URL=your_slack_webhook

# Integration APIs

SERVICENOW_USERNAME=automation_user
SERVICENOW_PASSWORD=your_password

# Maintenance System
MAINTENANCE_API_URL=https://api.maintenance-system.com
MAINTENANCE_API_KEY=your_api_key
```
